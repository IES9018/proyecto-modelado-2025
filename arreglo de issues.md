# Arreglo de Issues

## Documentación del proceso de creación de los diagramas

### 1. Relevamiento inicial

Para comenzar con los diagramas solicitados, se realizó primero un
relevamiento completo de los requisitos de la aplicación: - Se
identificaron los módulos principales. - Se revisó el flujo de datos
entre cada componente. - Se definieron las entidades que forman parte
del sistema.

### 2. Construcción del Diagrama de Casos de Uso

Se listaron todos los actores involucrados y las interacciones
posibles: - Usuario anónimo - Usuario registrado - Administrador

Luego, se establecieron los casos de uso principales: - Leer libros -
Consultar personajes - Consultar mapas y lugares - Registrar cuenta -
Administrar contenido

Finalmente, se organizó la estructura visual considerando relaciones
**include**, **extend** y herencia entre actores.

### 3. Construcción del Diagrama de Clases

En esta etapa se: 1. Identificaron las clases base:\
- Libro\
- Personaje\
- Lugar\
- Mapa\
- Usuario\
2. Se determinaron atributos y métodos relevantes.\
3. Se definieron relaciones entre clases:\
- Dependencias\
- Agregaciones\
- Composiciones\
- Herencias

La estructura final permitió visualizar cómo se relacionan los datos del
sistema.

### 4. Validación del modelo

Se revisaron los diagramas comparándolos con los requisitos iniciales.
Esto permitió: - Ajustar clases que faltaban.\
- Eliminar relaciones redundantes.\
- Asegurar consistencia conceptual.

### 5. Conclusión

Este documento resume el proceso seguido para generar los diagramas del
sistema, garantizando claridad y trazabilidad desde la identificación
inicial de requisitos hasta el modelado visual final.
